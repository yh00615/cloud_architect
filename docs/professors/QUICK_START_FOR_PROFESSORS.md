## 시작하기 전에

- ✅ **Kiro 에디터** (https://kiro.ai 에서 다운로드)
- ✅ **프로젝트 폴더** (개발팀에서 제공)

**❌ 수정 불가** (개발팀 영역):

## 5분 빠른 시작

### 1단계: Kiro 열기 (30초)

1. **Kiro 실행**
   - macOS: Applications 폴더에서 Kiro 더블클릭
   - Windows: 시작 메뉴에서 Kiro 검색

2. **프로젝트 열기**
   - 메뉴: **File > Open Folder**
   - `university-lab-guide` 폴더 선택
   - **Open** 클릭

3. **로딩 대기**
   - 프로젝트가 로드될 때까지 10초 정도 대기

### 2단계: 가이드 파일 찾기 (1분)

1. **왼쪽 파일 탐색기에서 폴더 열기**
   ```
   public/
   └── content/
       └── week1/
           └── 1-1-well-architected-tool-demo.md
   ```

2. **파일 클릭**
   - `1-1-well-architected-tool-demo.md` 클릭
   - 오른쪽에 파일 내용이 표시됨

### 3단계: 내용 수정하기 (2분)

**예시: 제목 수정**

**수정 전**:
```markdown
---
title: "AWS Well-Architected Tool 워크로드 평가"
---
```

**수정 후**:
```markdown
---
title: "AWS Well-Architected Tool 워크로드 평가 (2024년 업데이트)"
---
```

**예시: 단계 추가**

**수정 전**:
```markdown
1. AWS Management Console에 로그인합니다.
2. Well-Architected Tool을 검색합니다.
```

**수정 후**:
```markdown
1. AWS Management Console에 로그인합니다.
2. 상단 검색창에서 `Well-Architected Tool`을 검색합니다.
3. 검색 결과에서 **AWS Well-Architected Tool**을 선택합니다.
```

### 4단계: 저장하기 (30초)

1. **파일 저장**
   - macOS: `Cmd + S`
   - Windows: `Ctrl + S`

2. **저장 확인**
   - 파일 탭에서 점(●)이 사라지면 저장 완료

### 5단계: 변경사항 확인 (1분)

   - 또는 메뉴: **View > Terminal**

2. **개발 서버 시작**

3. **브라우저에서 확인**

   - 주소창에 `http://localhost:3000` 입력

## 자주 하는 작업

### 작업 1: 새로운 실습 단계 추가

**위치**: 태스크 섹션 내부

**예시**:
```markdown
## 태스크 1: S3 버킷 생성

1. AWS Management Console에 로그인합니다.
2. 상단 검색창에서 `S3`를 검색하고 선택합니다.
3. [[Create bucket]] 버튼을 클릭합니다.
4. **Bucket name**에 `my-bucket-12345`를 입력합니다.
5. [[Create bucket]] 버튼을 클릭합니다.

✅ **태스크 완료**: S3 버킷이 생성되었습니다.
```

**팁**:
- 각 단계는 하나의 구체적인 행동만 포함
- AWS 버튼은 `[[버튼명]]` 형식 사용
- 입력값은 `` `값` `` 형식 사용
- 완료 메시지는 `✅ **태스크 완료**:` 형식 사용

### 작업 2: 경고 메시지 추가

**문법**: `> [!WARNING]`

**예시**:
```markdown
> [!WARNING]
> 이 실습에서 생성하는 리소스는 실습 종료 후 반드시 삭제해야 합니다.
> EC2 인스턴스(t2.micro)는 시간당 약 $0.0116이 부과됩니다 (ap-northeast-2 리전 기준).
```

**다른 Alert 타입**:
- `> [!NOTE]` - 참고 사항 (파란색)
- `> [!TIP]` - 팁 (초록색)
- `> [!IMPORTANT]` - 중요 (보라색)

### 작업 3: 실습 파일 추가

**단계**:

1. **파일 준비**
   - 실습에 필요한 파일 생성 (예: `template.yaml`)
   - ZIP 파일로 압축 (예: `week2-1-files.zip`)

2. **파일 업로드**
   - Kiro에서 `public/files/week2/` 폴더 열기
   - ZIP 파일을 폴더에 드래그 앤 드롭

3. **다운로드 블록 추가**
   ```markdown
   > [!DOWNLOAD]
   > [week2-1-iam-policies.zip](/files/week2/week2-1-iam-policies.zip)
   > - `policy1.json` - S3 읽기 전용 정책
   > - `policy2.json` - S3 쓰기 권한 정책
   > - `README.md` - 정책 사용 방법
   ```

### 작업 4: 학습 목표 수정

**위치**: Front Matter (파일 상단)

**예시**:
```markdown
---
title: "IAM 정책 생성 및 관리"
week: 2
session: 1
awsServices:
  - AWS IAM
  - Amazon S3
learningObjectives:
  - IAM 정책을 생성할 수 있습니다
  - 정책을 사용자에게 연결할 수 있습니다
  - 정책 효과를 테스트할 수 있습니다
prerequisites:
  - AWS 계정 보유
  - IAM 기본 개념 이해
---
```

**수정 방법**:
- `learningObjectives:` 아래에 항목 추가/수정
- 각 항목은 `- `로 시작
- "~할 수 있습니다" 형식 사용

### 작업 5: 변경사항 저장 (Git)

**Kiro에서 Git 사용**:

1. **변경된 파일 확인**
   - 왼쪽 사이드바에서 Git 아이콘 클릭 (세 번째 아이콘)
   - 변경된 파일 목록 확인

2. **커밋 메시지 작성**
   - 상단 입력창에 메시지 입력
   - 예: "Week 2-1 IAM 정책 가이드 업데이트"

3. **커밋 및 푸시**
   - **Commit** 버튼 클릭
   - **Push** 버튼 클릭 (또는 Sync)

**커밋 메시지 예시**:
- "Week 1-1 제목 수정"
- "Week 2-1 실습 단계 추가"
- "Week 3-1 경고 메시지 추가"
- "Week 4-1 실습 파일 업데이트"

## 문제 해결

### 문제 1: 파일을 저장했는데 변경사항이 보이지 않아요

**해결**:
1. 브라우저에서 페이지 새로고침 (`Cmd+R` 또는 `Ctrl+R`)
2. 개발 서버가 실행 중인지 확인 (터미널에서 `npm run dev`)
3. 올바른 파일을 수정했는지 확인

### 문제 2: 버튼이 오렌지색으로 표시되지 않아요

**원인**: 버튼 문법 오류

**잘못된 예시**:
```markdown
[Create bucket] 버튼을 클릭합니다.  ❌
```

**올바른 예시**:
```markdown
[[Create bucket]] 버튼을 클릭합니다.  ✅
```

**규칙**:
- Primary 버튼: `[[버튼명]]` (대괄호 2개)
- Normal 버튼: `{{버튼명}}` (중괄호 2개)
- Link 버튼: `((버튼명))` (소괄호 2개)

### 문제 3: 복사 버튼이 나타나지 않아요

**원인**: 백틱(`) 누락

**잘못된 예시**:
```markdown
Bucket name에 my-bucket을 입력합니다.  ❌
```

**올바른 예시**:
```markdown
Bucket name에 `my-bucket`을 입력합니다.  ✅
```

### 문제 4: Alert 박스가 표시되지 않아요

**원인**: Alert 문법 오류

**잘못된 예시**:
```markdown
> WARNING
> 주의하세요.  ❌
```

**올바른 예시**:
```markdown
> [!WARNING]
> 주의하세요.  ✅
```

**규칙**:
- `>` 다음에 공백 필요
- `[!타입]` 형식 사용
- 타입: WARNING, NOTE, TIP, IMPORTANT

### 문제 5: Git Push가 실패해요

**원인**: 권한 문제 또는 충돌

**해결**:
1. **Pull 먼저 실행**
   - Git 패널에서 **Pull** 버튼 클릭
   - 다른 사람의 변경사항 가져오기

2. **충돌 해결** (충돌 발생 시)
   - Kiro가 충돌 파일 표시
   - 충돌 부분 수동 수정
   - 다시 커밋 및 푸시

3. **개발팀에 문의**
   - 권한 문제인 경우 개발팀에 연락

### 문제 6: 변경사항이 웹사이트에 반영되지 않아요

**원인**: Git Push 후 배포 대기 시간

> [!NOTE]
> 배포 방법이 결정되면 아래 단계가 업데이트됩니다.

<!-- TODO: 배포 플랫폼별 상세 문제 해결 가이드 작성
각 플랫폼별 추가 필요 정보:
1. 배포 로그 확인 방법
2. 배포 실패 시 알림 방법
3. 수동 재배포 방법
4. 캐시 무효화 방법
5. 긴급 롤백 방법

**해결**:
1. **Git Push 확인**
   - Git 패널에서 Push가 완료되었는지 확인
   - "✓ Pushed" 메시지 확인

2. **배포 대기**
   - 배포 플랫폼에서 자동 배포 진행 중
   - 예상 시간: 2-5분 (플랫폼에 따라 다름)

3. **배포 상태 확인**
   - 배포 대시보드: `[배포 상태 확인 URL - 추후 업데이트]`
   - 배포 완료 후 웹사이트 새로고침

4. **캐시 문제 해결** (배포 완료 후에도 반영 안 될 때)
   - 브라우저 강력 새로고침: `Cmd+Shift+R` (Mac) 또는 `Ctrl+Shift+R` (Windows)
   - 시크릿 모드에서 확인

> [!IMPORTANT]
> 배포 관련 문제는 개발팀에 문의하세요.
> - 개발팀 이메일: `[이메일 - 추후 업데이트]`
> - 긴급 연락: `[전화번호 - 추후 업데이트]`

### 문제 7: 파일 업로드가 안 돼요

**원인**: 파일 크기 또는 권한 문제

**해결**:
1. **파일 크기 확인**
   - ZIP 파일: 최대 50MB
   - 이미지: 최대 5MB
   - 너무 크면 압축하거나 분할

2. **파일 권한 확인**
   - 파일이 읽기 전용인지 확인
   - 필요시 권한 변경

3. **드래그 앤 드롭 대신 복사**
   - Finder/탐색기에서 파일 복사
   - Kiro 폴더에 붙여넣기
