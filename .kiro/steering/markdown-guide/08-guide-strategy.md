# 가이드 작성 전략 (Guide Strategy)

## 개요

실습 가이드와 데모 가이드의 작성 전략, 태스크 구성, 복잡도 관리 방법을 설명합니다.
효과적인 가이드 작성을 위한 핵심 원칙과 실전 전략을 제공합니다.

---

## 🎯 핵심 원칙

### 실습과 데모는 목적과 사용 방식이 완전히 다릅니다

#### 30분 차시 구성 (PPT 촬영본)

- **30분 = PPT 이론 강의 + 데모 시연** (교수가 촬영)
- **데모 가이드 = 촬영본에서 시연할 내용**
- **목적**: 교수가 개념을 설명하면서 실제 동작을 보여주기 위한 시나리오

#### 실습 가이드 (별도 진행)

- **실습 가이드 = 학생이 직접 따라하는 실습**
- **시간 제한 없음** (학생이 자기 속도로 진행)
- **목적**: 학생이 막히지 않고 따라할 수 있도록 상세한 단계별 지침 제공

---

## 📋 가이드 작성 원칙

### 실습 가이드 (Lab)

- 학생이 **직접 실습**하는 내용
- 태스크 0 (환경 구축): CloudFormation으로 복잡한 사전 인프라 자동 생성
- 핵심 태스크: **해당 차시에서 꼭 배워야 할 것만** 포함
- 리소스 정리: CloudFormation 스택 삭제
- **원칙**: 복잡한 환경 구축은 태스크 0으로, 학습 목표는 핵심 태스크로

### 데모 가이드 (Demo)

- 교수가 **PPT 촬영본에서 시연**하는 내용
- 태스크 0 (환경 구축): CloudFormation으로 복잡한 사전 인프라 자동 생성 (필요시)
- 개념 설명 + 시연: 이론과 실습을 병행하여 촬영
- 리소스 정리: CloudFormation 스택 삭제 (필요시)
- **원칙**: 30분 촬영본에 들어갈 핵심 개념과 시연만 포함

---

## 🏗️ 태스크 0: 환경 구축 표준

### 태스크 0이 필요한 경우

다음과 같은 **복잡한 사전 환경 구축**이 필요하면 태스크 0으로 자동화:

- ✅ VPC, 서브넷, 보안 그룹 등 네트워크 인프라
- ✅ EC2 인스턴스, RDS, ElastiCache 등 사전 리소스
- ✅ IAM 역할, S3 버킷 등 권한 및 스토리지 설정
- ✅ 학습 목표와 직접 관련 없는 반복적인 설정 작업

**핵심 원칙**: 
- **학습 목표와 무관한 복잡한 설정은 태스크 0으로 자동화**
- **학습 목표에 집중할 수 있도록 사전 준비**
- 예: VPC Endpoint 실습이 목표라면 → VPC/서브넷/EC2는 태스크 0으로 자동 생성

### 태스크 0 작성 규칙

1. CloudFormation 템플릿 제공 (`public/files/week{X}/`)
2. 출력값 명시 (Outputs 섹션)
3. 대기 시간 안내 (NOTE Alert)
4. 다음 태스크 연결 (IMPORTANT Alert)

---

## 📊 태스크 구성 가이드라인

### 권장사항 (엄격한 규칙 아님)

- 태스크 수: 3-5개 정도 (태스크 0 포함)
- 각 태스크: 핵심 학습 목표에 집중
- 단계 수: 태스크당 적절한 분량 (너무 많거나 적지 않게)

### 핵심 원칙

- ✅ **해당 차시에 꼭 배워야 할 것만 포함**
- ✅ **사전 환경 구축은 CloudFormation으로 자동화**
- ✅ **복잡한 설정은 미리 만들어두고 학습 목표에 집중**
- ❌ 불필요하게 많은 단계로 학생을 지치게 하지 않기

### 표준 구조

- 태스크 0: 실습 환경 구축 (CloudFormation)
- 태스크 1-N: 핵심 작업 (3-7개 단계)
- 리소스 정리: CloudFormation 스택 삭제

---

## 🎯 복잡도 관리 원칙

- 복잡한 환경 구축 → 태스크 0 (CloudFormation)
- 핵심 학습 목표 → 태스크 1, 2, 3...
- 검증 및 테스트 → 마지막 태스크
- 리소스 정리 → CloudFormation 스택 삭제

---

## 🎭 데모 가이드 작성 전략

- CONCEPT Alert 적극 활용 (태스크당 1-3개)
- 📚 참고 섹션 필수
- 실습과 동일한 상세도 (클릭 단위 지침)
- 교수가 시연할 수 있도록 명확한 단계

---

## ✅ 가이드 작성 체크리스트

### 실습 가이드 체크리스트

- [ ] **학습 목표 명확**: 이 차시에서 꼭 배워야 할 것이 무엇인가?
- [ ] **태스크 0 활용**: 복잡한 환경 구축은 CloudFormation으로 자동화했는가?
- [ ] **핵심에 집중**: 학습 목표와 무관한 반복 작업은 제거했는가?
- [ ] **파일 제공**: DOWNLOAD Alert로 CloudFormation 템플릿 및 실습 파일 제공
- [ ] **매우 상세한 단계**: 학생이 막히지 않도록 클릭 단위 지침
- [ ] **CONCEPT Alert**: 필요시 사용 (복잡한 개념 설명), 연속 사용 금지
- [ ] **리소스 정리**: CloudFormation 스택 삭제로 간단하게

### 데모 가이드 체크리스트

- [ ] **촬영 시나리오**: 교수가 30분 내 시연 가능한 내용인가?
- [ ] **이론 연계**: PPT 이론 강의와 자연스럽게 연결되는가?
- [ ] **개념 설명**: CONCEPT Alert 적극 활용, 연속 사용 금지
- [ ] **📚 참고 섹션**: 필수 포함 (심화 개념 설명)
- [ ] **태스크 0 활용**: 복잡한 환경 구축은 CloudFormation으로 자동화 (필요시)
- [ ] **핵심 개념**: 30분 촬영본에 들어갈 핵심 개념만 포함

### 각 태스크

- [ ] 단계 수: 3-7개
- [ ] 각 단계는 하나의 구체적인 행동만 포함
- [ ] Alert 사용: 태스크당 3개 이하
- [ ] 완료 확인 메시지 포함

### 복잡도

- [ ] 복잡한 환경 구축은 태스크 0으로 이동
- [ ] CloudFormation으로 자동화 가능한 부분은 템플릿으로 제공
- [ ] 핵심 학습 목표에 집중

## 📊 실습 vs 데모 최종 정리

| 구분 | 실습 가이드 | 데모 가이드 |
|------|------------|------------|
| **목적** | 학생이 직접 실습하여 기술 습득 | 교수가 PPT 촬영본에서 개념 설명 + 시연 |
| **사용 방식** | 학생이 자기 속도로 따라함 | 교수가 30분 영상에서 시연 |
| **시간 제약** | ❌ 없음 (학생 속도에 맞춤) | ✅ 가능하면 30분 내외 (이론 + 데모), 필요시 초과 가능 |
| **상세도** | 매우 상세 (클릭 단위) | 매우 상세 (클릭 단위) |
| **태스크 0** | CloudFormation 환경 구축 (필요시) | CloudFormation 환경 구축 (필요시) |
| **태스크 수** | 유연하게 조정 (보통 3-4개 권장) | 유연하게 조정 (보통 2-4개 권장) |
| **CONCEPT Alert** | 필요시 사용 (복잡한 개념) | 적극 활용 |
| **📚 참고 섹션** | 선택사항 | 필수 |
| **파일 제공** | ✅ DOWNLOAD Alert | ❌ 제공 안 함 |
| **촬영** | ❌ 촬영 안 함 | ✅ 교수가 PPT와 함께 촬영 |

## 📝 작성 순서

1. 학습 목표 정의
2. 태스크 분할 (태스크 0 필요시 포함)
3. CloudFormation 템플릿 작성
4. 각 태스크 작성 (3-7개 단계)
5. Alert 추가 (과다 사용 금지)
6. 리소스 정리

## 🎯 작성 시 주의사항

- 데모: 교수가 촬영하면서 설명할 수 있도록, CONCEPT Alert 적극 활용

## 🌟 친절도 핵심 원칙

실습 가이드는 AWS 공식 실습 가이드 수준의 친절도를 목표로 합니다.

### 핵심 7가지 요소

1. **대기 시간 활용 제안**: 무엇이 일어나는지 설명 + 대기 중 할 일 제안
2. **상태 변화 완전 설명**: 모든 가능한 상태 + 색상 + 의미 + 실패 시 대처
3. **UI 요소 위치 설명**: 화면 이동 + 구조 + 어디를 봐야 하는지 명시
4. **출력값 복사 상세 안내**: Key/Value 열 구조 + 복사 아이콘 위치 + 예시
5. **문제 해결 사전 안내**: 정상 동작 + 오류 가능성 + 원인 + 해결 방법
6. **단계 극도로 세분화**: 페이지 열림 확인 + 스크롤 + 찾기와 클릭 분리
7. **예시 값 구체적 제공**: 값의 형태 예시 (vpc-xxx, i-xxx) + [!OUTPUT] Alert

---

## 📋 친절도 체크리스트

- [ ] 대기 시간: 무엇이 일어나는지 + 할 일 제안
- [ ] 상태 변화: 모든 상태 + 색상 + 의미 + 실패 대처
- [ ] UI 요소: 화면 위치 + 구조 설명
- [ ] 출력값: Key/Value 구조 + 예시 + 사용처
- [ ] 문제 해결: 정상 + 오류 + 해결
- [ ] 단계 세분화: 페이지 확인 + 스크롤 + 찾기/클릭 분리
- [ ] 예시 값: vpc-xxx, i-xxx 형태 제공
