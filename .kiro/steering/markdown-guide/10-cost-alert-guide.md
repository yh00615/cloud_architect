# 비용 Alert 가이드 (Cost Alert Guide)

## 개요

[!COST] Alert는 실습에서 생성하는 AWS 리소스의 예상 비용을 학생들에게 안내하는 교육용 Alert입니다.
대학생 학습 맥락에 맞춰 부담 없이 비용 정보를 제공하면서, 실무에서 필요한 비용 관리 감각을 기를 수 있도록 설계되었습니다.

---

## 사용 목적

### 교육적 목적

- AWS 리소스별 비용 구조 이해
- 실무에서 필요한 비용 산정 능력 향상
- 클라우드 비용 최적화 개념 학습

### 실용적 목적

- 개인 계정 사용 시 예상 비용 파악
- 실습 후 리소스 정리의 중요성 인식
- 프리티어 범위 확인

---

## Alert 타입

### [!COST] - 리소스 비용

- **색상**: 연한 파란색 (info)
- **아이콘**: `status-info`
- **용도**: 리소스별 시간당/월간 비용 안내
- **톤**: 교육적, 정보 제공

---

## 표준 형식

### 기본 형식 (필수 요소)

```markdown
> [!COST]
> **리소스 비용 (ap-northeast-2 기준)**
>
> - 리소스명 (타입): 시간당/월간 비용
> - 리소스명 (타입): 시간당/월간 비용
> - 예상 실습 시간: X시간
> - **예상 총 비용**: 약 $X.XX
>
> 리전별로 요금이 다를 수 있습니다. 최신 요금은 [AWS 요금 페이지](https://aws.amazon.com/pricing/)에서 확인하세요.
```

### 필수 포함 요소

1. **리전 명시**: `(ap-northeast-2 기준)` 또는 `(us-east-1 기준)`
2. **리소스별 비용**: 불릿 포인트로 나열
3. **예상 총 비용**: 굵은 글씨로 강조
4. **리전별 차이 안내**: 마지막 문장에 필수 포함
5. **AWS 요금 페이지 링크**: 최신 정보 확인 유도

## 작성 규칙

### 1. 리전 명시 (필수)

- 항상 리전을 명시: `(ap-northeast-2 기준)`, `(us-east-1 기준)`
- 실습에서 사용하는 리전과 일치해야 함

### 2. 비용 단위

- **시간당 비용**: `시간당 $0.017`
- **월간 비용**: `월 $12.50` (730시간 기준)
- **요청당 비용**: `100만 건당 $0.20`
- **데이터 전송**: `GB당 $0.09`

### 3. 비용 표기

- 소수점 3자리까지: `$0.017`
- 총 비용은 2자리: `$0.05`
- 범위 표기: `$0.05-0.10`

### 4. 프리티어 안내

- 프리티어 범위 내: `(프리티어: 5GB 무료)`
- 프리티어 초과 가능성: `프리티어 한도를 초과할 수 있습니다`

### 5. 실무 팁 추가 (선택)

- 고비용 리소스: 대안 제시
- 비용 최적화: 실무 모범 사례
- 모니터링: AWS Cost Explorer, Budgets 소개

## 배치 위치

### 실습 가이드

- **Front Matter 직후**: 전체 실습 비용 안내
- **태스크 시작 전**: 특정 리소스 생성 전 비용 안내
- **리소스 정리 섹션**: 비용 절감 중요성 강조

### 데모 가이드

- **개요 섹션**: 데모에서 사용하는 리소스 비용
- **고비용 리소스 생성 전**: 사전 안내

## 금지 사항

### ❌ 하지 말아야 할 것

- 경고(WARNING) 톤 사용 금지 → 정보(INFO) 톤 사용
- 과도한 비용 강조 금지 → 교육적 맥락 유지
- 정확하지 않은 비용 정보 금지 → AWS 공식 요금 페이지 확인
- 리전 명시 누락 금지 → 항상 리전 포함
- "리전별 차이" 문구 누락 금지 → 필수 포함

### ❌ 잘못된 예시

```markdown
> [!WARNING]
> 이 실습은 비용이 많이 발생합니다! 주의하세요!
```

```markdown
> [!COST]
>
> - RDS: 약간의 비용 발생
> - 총 비용: 조금
```

## 실무 연계 팁

### 학생들에게 전달할 메시지

1. **비용 인식**: 클라우드 리소스는 사용한 만큼 비용이 발생합니다
2. **리소스 정리**: 실습 후 반드시 리소스를 삭제하세요
3. **비용 모니터링**: AWS Cost Explorer로 실시간 비용 확인
4. **예산 설정**: AWS Budgets로 예산 초과 알림 설정
5. **최적화**: 적절한 인스턴스 타입과 리전 선택
