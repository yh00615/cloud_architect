# 표준 문구 (Standard Phrases)

실습 가이드 작성 시 반드시 준수해야 할 표준 표현 규칙입니다.

---

## 필수 표준 문구 (50개 규칙)

### 1. AWS 서비스 접근
- `AWS Management Console에 로그인한 후 상단 검색창에서 `서비스명`을 검색하고 선택합니다.`

### 2. 왼쪽 메뉴 선택
- `왼쪽 메뉴에서 **메뉴명**을 선택합니다.`
- 예외: 새 콘솔 이동 직후만 콘솔 이름 명시 가능

### 3. 콘솔 이동
- `~로 이동합니다` (처음/재방문 모두 통일)

### 4. 문장 종결
- 모든 단계는 마침표(`.`)로 종결

### 5. 괄호 설명 제거
- 불필요한 괄호 설명 제거
- 중요 정보는 별도 NOTE로 작성

### 6. 버튼 클릭
- `[[버튼명]] 버튼을 클릭합니다`

### 7. 필드 입력
- `**필드명**에 `값`을 입력합니다`

### 8. 파일 압축 해제
- `다운로드한 {파일명} 파일의 압축을 해제합니다`

### 9. 확인 표현
- 단순: `~를 확인합니다`
- 조건: `~가 선택되어 있는지 확인합니다`
- 가능성: `~를 확인할 수 있습니다`
- 명령형만 사용 (청유형 금지)

### 10. 대기 시간
- 단계: 간결하게 (`상태가 "Available"로 변경될 때까지 기다립니다`)
- 시간 정보: NOTE Alert로 분리

### 11. 상태 변경
- `상태가 "{이전상태}"에서 "{새상태}"로 변경될 때까지 기다립니다`
- 상태값은 큰따옴표로 감싸기

### 12. 복사 및 저장
- 단순: `~를 복사합니다`
- 저장: `~를 복사하여 메모장에 저장합니다`
- 나중 사용 시 NOTE로 안내

### 13. 탭 선택
- `**탭명** 탭을 선택합니다`
- 항상 "선택합니다"로 통일

### 14. 새 브라우저 탭
- `새 브라우저 탭을 엽니다`

### 15. 드롭다운 선택
- `**필드명**에서 `값`을 선택합니다`

### 16. Actions 메뉴
- `**Actions** > `메뉴항목`을 선택합니다`

### 17. 체크박스
- `~를 체크합니다`
- 체크박스: "체크", 라디오/드롭다운: "선택"

### 18. URL 접속
- `주소창에 붙여넣고 Enter를 누릅니다`
- 간결: `새 브라우저 탭을 열고 복사한 도메인으로 접속합니다`

### 19. 탭 선택 표준화
- 상단 탭: `**탭명** 탭을 선택합니다`
- 하단 탭: `하단의 **탭명** 탭을 선택합니다`

### 20. 연속 동작
- `~한 후 ~합니다` ("~하고" 금지)

### 21. 단계 번호 중복 방지
- 각 단계는 고유 번호, 연속 증가

### 22. 페이지 새로고침
- `페이지를 새로고침합니다`

### 23. 명령어 실행
- `다음 명령어를 실행합니다:` (콜론 필수)

### 24. 스크롤
- `아래로 스크롤하여 ~를 확인합니다`

### 25. 리소스 선택
- `~를 선택합니다`
- 버튼: "클릭", 리소스: "선택"

### 26. 값 확인 및 검증
- `~인지 확인합니다`
- `~가 ~인지 확인합니다`

### 27. 섹션/영역 이동
- `~섹션에서` / `~영역에서`

### 28. 설정 유지
- `기본값을 유지합니다`
- `~를 선택한 상태로 유지합니다`

### 29. 메모장 저장
- `~를 복사하여 메모장에 저장합니다`
- NOTE로 사용 시점 안내

### 30. 다운로드 파일 설명
- 파일 설명: 5-50자 (파일명 + 간단 용도)
- 관련 태스크: 30자 이상 (파일명 + 구체적 사용법)

### 31. 청유형 완전 금지
- "~하세요", "~해주세요" 금지
- "~합니다" 명령형만 사용
- Alert 내부도 예외 없음

### 32. 콘솔 재방문 통일
- 처음/재방문 모두 "~로 이동합니다"
- "돌아갑니다", "다시 이동합니다" 금지

### 33. 용어 표기
- AWS 서비스명: 영어 필수 (Amazon S3, AWS Lambda)
- AWS 콘솔 UI: 영어 유지 (Create function, Actions)
- 일반 IT 용어: 한글/영어 선택 후 문서 내 일관성 유지
- 혼용 금지

### 34. 컨테이너/Pod 접속
- 접속: `~에 접속합니다` / `~내부에 접속합니다`
- 종료: `~에서 나옵니다`

### 35. 웹 브라우저 접속
- `웹 브라우저에서 ~로 접속하여 ~를 확인합니다`

### 36. Lambda 함수 탭 이동
- "~탭으로 돌아갑니다" 금지
- "~콘솔로 이동합니다" + "함수를 선택합니다"

### 37. Configuration 탭 하위 메뉴
- 탭 선택과 하위 메뉴 선택을 별도 단계로 분리

### 38. AWS 리전 선택
- `**AWS Region**에서 `Asia Pacific (Seoul) ap-northeast-2`를 선택합니다.`
- 전체 리전명 + 코드 필수

### 39. 태스크 제목
- `## 태스크 X: 작업 내용을 명확히 설명하는 문장`
- 괄호 설명 금지

### 40. Lambda 함수 코드 주석
- 모듈 레벨 DocString 필수
- 함수 레벨 DocString 필수
- 한국어 주석 필수

### 41. 함수 파라미터 표 형식
- 표 형식 필수 (파라미터 | 타입 | 필수 | 설명)
- 필수 여부: ✅/❌ 이모지

### 42. 코드 블록 언어 명시
- 모든 코드 블록에 언어 명시 필수

### 43. 아키텍처 다이어그램
- 이미지 + 구성 요소 + 데이터 흐름 모두 포함

### 44. 환경 변수
- 표 형식 필수 (변수명 | 값 | 설명)

### 45. 문제 해결 섹션
- 문제 + 증상 + 원인 + 해결 순서

### 46. 비용 정보
- 표 형식 (리소스 | 타입 | 시간당 비용)
- 리전 명시 필수

### 47. OUTPUT Alert
- 반드시 코드 블록 사용

### 48. DynamoDB 명명 규칙
- camelCase 사용 (`userId`, `orderId`, `createdAt`)
- 상태값: 소문자 String (`delivered`, `pending`)
- 타임스탬프: ISO 8601 전체 형식
- 예외: Python Lambda는 snake_case 허용

### 49. AWS 서비스 표시 정책
- 핵심 서비스만 표시 (학습 목표)
- 보조 서비스 제외 (VPC, EC2, IAM)

### 50. 샘플 데이터 일관성
- 값 차이 허용 (ID, 날짜, 금액)
- 명명 규칙 불일치 불허용
