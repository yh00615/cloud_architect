# 페이지 구조 표준

실습 가이드와 데모 가이드의 표준 구조 및 필수 섹션입니다.

---

## 실습 가이드 (Lab) 구조

### 필수 구성 순서

1. **Front Matter** (메타데이터)
2. **실습 개요** (1-2문단)
3. **[!DOWNLOAD]** (파일 제공 시)
4. **[!WARNING]** (비용 경고)
5. **태스크 0**: 사전 환경 구축 (CloudFormation)
6. **태스크 1-N**: 핵심 실습
7. **마무리** (완료 항목 요약)
8. **리소스 정리** (CloudFormation 스택 삭제)
9. **추가 학습 리소스**
10. **[선택] 사전 환경 직접 구축**
11. **[선택] 📚 참고**

### Front Matter 필수 필드

```yaml
---
title: "실습 제목"
week: 주차번호
session: 세션번호
awsServices:
  - AWS 서비스 (핵심만)
learningObjectives:
  - 학습 목표 1
prerequisites:
  - 사전 요구사항
---
```

### 태스크 구조

```markdown
## 태스크 X: [작업 내용 명확히]

### 태스크 설명 (필수)
[1-2문단, 무엇을/왜 하는지]

### 상세 단계 (필수)
1. 첫 번째 단계
2. 두 번째 단계

> [!NOTE] (필요시)
> 참고 사항

✅ **태스크 완료**: 완료 메시지
```

### 태스크 규칙
- 단계 수: 3-7개 권장
- Alert: 태스크당 최대 3개
- 각 단계: 하나의 구체적 행동만

## 데모 가이드 (Demo) 구조

### 실습과의 차이점

| 구분 | 실습 | 데모 |
|------|------|------|
| 파일 제공 | ✅ DOWNLOAD | ❌ 없음 |
| 개념 설명 | 최소화 | ✅ CONCEPT 적극 활용 |
| 📚 참고 | 선택 | 필수 |
| 학습 목표 | "~할 수 있습니다" | "~를 이해/설명" |

### 필수 구성

1. Front Matter
2. 개요
3. [!WARNING] (리소스 생성 시) / [!NOTE] (비용 없음)
4. 태스크 1-N (CONCEPT Alert 포함)
5. 마무리
6. 리소스 정리 (필요시)
7. 추가 학습 리소스
8. **📚 참고** (필수)

## 리소스 정리 표준

```markdown
## 리소스 정리

> [!WARNING]
> 다음 단계를 **반드시 수행**하여 불필요한 비용을 방지하세요.

### CloudFormation 스택 삭제

1. CloudFormation 콘솔로 이동합니다.
2. `week{X}-{Y}-lab-stack` 스택을 선택합니다.
3. [[Delete]] 버튼을 클릭합니다.
4. 확인 창에서 [[Delete]] 버튼을 클릭합니다.
5. 스택 삭제가 완료될 때까지 기다립니다.

> [!NOTE]
> CloudFormation 스택을 삭제하면 모든 리소스가 자동으로 삭제됩니다.

✅ **실습 종료**: 모든 리소스가 정리되었습니다.
```
