# 📚 참고 섹션 작성 표준 (Reference Section)

## 개요

📚 참고 섹션은 실습이나 데모에서 다룬 개념을 심화 설명하는 섹션입니다. 
- 실습 가이드: 선택사항 (복잡한 개념만)
- 데모 가이드: 필수

---

## 섹션 제목 규칙

```markdown
## 📚 참고: [주제명]
```

- 반드시 `📚` 이모지 포함
- 형식: `## 📚 참고: [주제명]`
- 주제명은 실습/데모에서 다룬 핵심 개념

---

## 하위 섹션 구조

- `###` 레벨: 주요 개념 카테고리
- `**굵은 글씨**`: 세부 개념이나 용어
- 불릿 포인트: 설명 항목
- 간결하고 명확한 문장 사용

---

## 이론 설명 패턴 (6가지)

### 1️⃣ 개념 비교 패턴
- 제목: `### [개념 A] vs [개념 B]`
- 각 개념을 `**굵은 글씨**:`로 시작
- 불릿 포인트로 특징 나열 (3-5개)

### 2️⃣ 구성 요소 설명 패턴
- 제목: `### 아키텍처 구성 요소`
- 각 구성 요소: `**이름 (영문명)**`
- 불릿 포인트로 역할과 특징 설명

### 3️⃣ 프로세스/단계 설명 패턴
- 제목: `### [프로세스명] 프로세스`
- 각 단계: `**X단계: [설명]**`
- 코드 예시 포함 가능

### 4️⃣ 사용 사례 패턴
- 제목: `### 사용 사례`
- 각 사례: `**사례명:**`
- 실무 적용 가능한 구체적인 예시

### 5️⃣ 모범 사례 패턴
- 제목: `### 보안 모범 사례`, `### 성능 최적화`
- 각 사례: `**원칙명:**`
- "~합니다" 형식 (명령형)

### 6️⃣ 개념 정의 패턴
- 제목: `### [용어명] ([영문명])`
- 첫 문장에서 명확한 정의 제공
- 필요시 불릿 포인트로 세부 설명

---

## 코드 예시 포함 규칙

- 코드 예시 전에 "**예시:**" 표시
- 코드 블록에 언어 명시 (json, bash, python 등)
- 실제 동작하는 코드 제공

---

## 문장 스타일 규칙

- 명령형 사용 ("~합니다")
- 청유형 금지 ("~하세요")
- 간결하고 명확하게
- 전문 용어는 한글(영문) 형식
