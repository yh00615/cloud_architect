# 이미지 및 스크린샷 가이드라인 (Image Guidelines)

## 개요

실습 가이드에서 이미지와 스크린샷을 효과적으로 사용하는 방법을 설명합니다.
적절한 이미지 사용은 학생의 이해도를 높이고 실습 진행을 원활하게 합니다.

---

## 📸 언제 이미지를 사용해야 하는가?

### ✅ 이미지 사용이 필요한 경우

1. **복잡한 UI 설명**
   - AWS 콘솔의 복잡한 설정 화면
   - 여러 옵션이 있는 드롭다운 메뉴
   - 찾기 어려운 버튼이나 메뉴 위치

2. **중요한 설정 화면**
   - 잘못 설정하면 실습이 실패하는 중요한 단계
   - 여러 필드를 동시에 확인해야 하는 화면
   - 기본값과 다른 값을 입력해야 하는 경우

3. **예상 결과 화면**
   - 명령어 실행 후 나타나는 출력 결과
   - 리소스 생성 완료 후 상태 확인 화면
   - 성공/실패 메시지 화면

4. **아키텍처 다이어그램**
   - 실습에서 구축할 전체 아키텍처
   - 리소스 간 연결 관계
   - 네트워크 구성도

### ❌ 이미지 사용이 불필요한 경우

1. **간단한 버튼 클릭**
   - 명확한 버튼명이 있는 경우 (예: [[Create bucket]])
   - 단순한 메뉴 선택

2. **텍스트로 충분한 설명**
   - 간단한 입력값 (예: `my-bucket-123`)
   - 명확한 단계별 지침

3. **반복적인 화면**
   - 이미 앞에서 설명한 동일한 화면
   - 패턴이 동일한 반복 작업

## 📐 이미지 캡처 방법

### 1. 캡처 범위

#### ✅ 올바른 캡처 - 필요한 부분만
```
┌─────────────────────────────────┐
│ [필요한 UI 요소만 캡처]          │
│                                 │
│ ┌─────────────────────────┐    │
│ │ Bucket name             │    │
│ │ [my-bucket-123        ] │    │
│ │                         │    │
│ │ Region                  │    │
│ │ [ap-northeast-2 ▼]     │    │
│ └─────────────────────────┘    │
└─────────────────────────────────┘
```

#### ❌ 잘못된 캡처 - 전체 화면
```
┌─────────────────────────────────────────────┐
│ [브라우저 전체 + 불필요한 여백 포함]         │
│                                             │
│  [너무 많은 정보로 학생이 혼란스러움]       │
│                                             │
│  [파일 크기도 불필요하게 커짐]              │
└─────────────────────────────────────────────┘
```

### 2. 중요 부분 강조

#### 방법 1: 빨간 박스
```
┌─────────────────────────────────┐
│ ┏━━━━━━━━━━━━━━━━━━━━━━━━━┓   │
│ ┃ Bucket name             ┃   │ ← 빨간 박스로 강조
│ ┃ [my-bucket-123        ] ┃   │
│ ┗━━━━━━━━━━━━━━━━━━━━━━━━━┛   │
│                                 │
│ Region                          │
│ [ap-northeast-2 ▼]             │
└─────────────────────────────────┘
```

#### 방법 2: 화살표
```
┌─────────────────────────────────┐
│                                 │
│ Bucket name                     │
│ [my-bucket-123        ] ← 여기! │
│                                 │
│ Region                          │
│ [ap-northeast-2 ▼]             │
└─────────────────────────────────┘
```

#### 방법 3: 번호 표시
```
┌─────────────────────────────────┐
│ ① Bucket name                   │
│   [my-bucket-123        ]       │
│                                 │
│ ② Region                        │
│   [ap-northeast-2 ▼]           │
└─────────────────────────────────┘
```

### 3. 개인정보 제거

#### ✅ 올바른 처리 - 모자이크
```
┌─────────────────────────────────┐
│ Account ID: ████████████        │ ← 모자이크 처리
│ Email: ████████@example.com     │ ← 모자이크 처리
│ ARN: arn:aws:iam::████:role/... │ ← 일부만 표시
└─────────────────────────────────┘
```

#### 제거해야 할 정보
- ✅ AWS 계정 ID (12자리 숫자)
- ✅ 이메일 주소
- ✅ Access Key / Secret Key
- ✅ 개인 이름
- ✅ 회사명 (민감한 경우)
- ✅ IP 주소 (공개 IP)

#### 표시해도 되는 정보
- ✅ 리전명 (ap-northeast-2)
- ✅ 서비스명 (S3, EC2 등)
- ✅ 리소스 타입 (vpc-xxx, subnet-xxx)
- ✅ 예시 버킷명 (my-bucket-123)

### 4. 일관된 브라우저 및 설정

#### 권장 브라우저
- **Chrome** (최신 버전) - 권장
- **Firefox** (최신 버전) - 허용

#### 브라우저 설정
- **확대/축소**: 100% (기본값)
- **언어**: 영어 (AWS 콘솔 UI)
- **테마**: 라이트 모드 (일관성)
- **확장 프로그램**: 비활성화 (깔끔한 화면)

#### 화면 해상도
- **권장**: 1920x1080 (Full HD)
- **최소**: 1366x768
- **최대**: 2560x1440 (4K는 축소 필요)

## 📁 파일 명명 규칙

### 표준 형식
```
{주차}-{세션}-step{단계}-{설명}.png
```

### 예시
```
✅ 올바른 예시:
- 1-1-step1-s3-search.png
- 1-1-step2-create-bucket.png
- 2-1-step3-iam-policy-editor.png
- 3-2-step5-security-group-rules.png

❌ 잘못된 예시:
- screenshot1.png (정보 부족)
- week1_session1_step1.png (언더스코어 사용)
- 1-1-s3검색.png (한글 사용)
- IMG_20250207_123456.png (의미 없는 이름)
```

### 명명 규칙 상세

#### 주차 (week)
- 형식: 숫자 1-15
- 예: `1`, `2`, `11`, `14`

#### 세션 (session)
- 형식: 숫자 1-3
- 예: `1`, `2`, `3`

#### 단계 (step)
- 형식: `step` + 숫자
- 예: `step1`, `step5`, `step12`

#### 설명 (description)
- 형식: 영어 소문자, 하이픈으로 연결
- 간결하고 명확하게 (2-4 단어)
- 예: `s3-search`, `create-bucket`, `security-group-rules`

## 🎨 이미지 최적화

### 1. 파일 형식

#### PNG (권장)
- **장점**: 무손실 압축, 텍스트 선명
- **용도**: UI 스크린샷, 다이어그램
- **확장자**: `.png`

#### JPG (선택)
- **장점**: 파일 크기 작음
- **용도**: 사진, 복잡한 이미지
- **확장자**: `.jpg` 또는 `.jpeg`

#### SVG (다이어그램)
- **장점**: 확대해도 선명, 파일 크기 작음
- **용도**: 아키텍처 다이어그램, 아이콘
- **확장자**: `.svg`

### 2. 파일 크기

#### 목표 크기
- **스크린샷**: 100-300KB
- **다이어그램**: 50-150KB
- **최대 허용**: 500KB

#### 크기 초과 시 해결 방법
1. **압축 도구 사용**
   - TinyPNG (https://tinypng.com/)
   - ImageOptim (macOS)
   - Squoosh (https://squoosh.app/)

2. **해상도 조정**
   - 최대 너비: 1200px
   - 최대 높이: 800px

3. **불필요한 부분 제거**
   - 여백 최소화
   - 중복 정보 제거

### 3. 이미지 해상도

#### 권장 해상도
```
┌─────────────────────────────────┐
│ 용도          │ 너비    │ 높이   │
├─────────────────────────────────┤
│ 전체 화면     │ 1200px  │ 800px  │
│ 부분 캡처     │ 800px   │ 600px  │
│ 작은 UI 요소  │ 600px   │ 400px  │
│ 아이콘/버튼   │ 400px   │ 300px  │
└─────────────────────────────────┘
```

#### 반응형 고려사항
- 모바일에서도 읽을 수 있도록 텍스트 크기 확인
- 최소 너비 320px에서도 가독성 유지
- 중요한 정보는 이미지 중앙에 배치

## 📝 이미지 사용 예시

### 예시 1: 복잡한 UI 설정

```markdown
## 태스크 1: VPC Endpoint 생성

1. VPC 콘솔로 이동합니다.
2. 왼쪽 메뉴에서 **Endpoints**를 선택합니다.
3. [[Create endpoint]] 버튼을 클릭합니다.
4. 다음 설정을 입력합니다:
   - **Name tag**: `s3-gateway-endpoint`
   - **Service category**: `AWS services`
   - **Services**: `com.amazonaws.ap-northeast-2.s3` (Type: Gateway)

![VPC Endpoint 생성 화면에서 서비스 이름과 타입 선택](/images/week3/3-3-step4-endpoint-settings.png)

5. **VPC**에서 태스크 0에서 생성한 VPC를 선택합니다.
```

```markdown
## 태스크 2: 엔드포인트 동작 확인

1. EC2 인스턴스에 접속합니다.
2. 다음 명령어를 실행합니다:

```bash
aws s3 ls --debug 2>&1 | grep -i endpoint

3. 출력 결과에서 VPC Endpoint URL을 확인합니다.

## 🔧 이미지 편집 도구

### 스크린샷 캡처 도구

#### macOS
- **기본 도구**: Cmd + Shift + 4 (영역 선택)
- **Skitch**: 주석 추가 가능
- **CleanShot X**: 전문가용 (유료)

#### Windows
- **기본 도구**: Win + Shift + S (Snipping Tool)
- **Greenshot**: 무료, 주석 기능
- **ShareX**: 오픈소스, 다양한 기능

#### 크로스 플랫폼
- **Flameshot**: 오픈소스, 주석 기능
- **Lightshot**: 간단한 사용법

### 이미지 편집 도구

#### 간단한 편집
- **Preview** (macOS): 기본 편집 기능
- **Paint** (Windows): 기본 편집 기능
- **GIMP**: 무료, 고급 기능

#### 주석 추가
- **Skitch**: 화살표, 박스, 텍스트
- **Annotate**: 온라인 도구
- **Markup Hero**: 온라인 도구

#### 압축 도구
- **TinyPNG**: 온라인, PNG/JPG 압축
- **ImageOptim** (macOS): 무손실 압축
- **Squoosh**: 온라인, 다양한 형식

## ✅ 이미지 체크리스트

실습 가이드에 이미지를 추가하기 전에 다음을 확인하세요:

### 캡처 전
- [ ] 이미지가 정말 필요한가? (텍스트로 충분하지 않은가?)
- [ ] 어떤 부분을 강조할 것인가?
- [ ] 브라우저 설정이 일관되는가? (100% 확대, 영어 UI)

### 캡처 후
- [ ] 필요한 부분만 캡처했는가? (전체 화면 금지)
- [ ] 개인정보가 제거되었는가? (계정 ID, 이메일 등)
- [ ] 중요 부분이 강조되었는가? (빨간 박스, 화살표)
- [ ] 이미지가 선명한가? (흐릿하지 않은가?)

### 파일 저장
- [ ] 파일명이 규칙을 따르는가? (`{주차}-{세션}-step{단계}-{설명}.png`)
- [ ] 올바른 폴더에 저장했는가? (`public/images/week{주차}/`)
- [ ] 파일 크기가 적절한가? (500KB 이하)
- [ ] 파일 형식이 적절한가? (PNG 권장)

### 마크다운 작성
- [ ] 대체 텍스트가 명확한가? (10-20 단어)
- [ ] 이미지 경로가 정확한가? (`/images/week{주차}/{파일명}`)
- [ ] 이미지 위치가 적절한가? (관련 단계 바로 다음)
- [ ] 이미지 설명이 필요한가? (NOTE Alert 추가)

## 🎯 모범 사례 요약

### DO (해야 할 것)
- ✅ 복잡한 UI는 이미지로 설명
- ✅ 필요한 부분만 캡처
- ✅ 중요 부분 강조 (빨간 박스, 화살표)
- ✅ 개인정보 제거 (모자이크)
- ✅ 일관된 브라우저 설정 (100% 확대, 영어 UI)
- ✅ 명확한 파일명 (`{주차}-{세션}-step{단계}-{설명}.png`)
- ✅ 파일 크기 최적화 (500KB 이하)
- ✅ 명확한 대체 텍스트 (10-20 단어)

### DON'T (하지 말아야 할 것)
- ❌ 전체 화면 캡처
- ❌ 불필요한 이미지 남발
- ❌ 개인정보 노출
- ❌ 흐릿한 이미지
- ❌ 의미 없는 파일명 (`screenshot1.png`)
- ❌ 과도한 파일 크기 (1MB 이상)
- ❌ 대체 텍스트 누락 또는 불명확
