# 태스크 구성 및 환경 설정

태스크 번호 매기기와 태스크 0 판단 기준입니다.

---

## 태스크 0 판단 기준

### 핵심 원칙: 학습 목표 = 가이드 제공

**learningObjectives에 명시된 작업**은 학생이 직접 수행합니다.

```yaml
learningObjectives:
  - VPC Endpoint를 생성하여 S3에 접근할 수 있습니다
```

**판단**:
- ✅ VPC Endpoint → 가이드 제공 (학습 목표)
- ✅ VPC, 서브넷, EC2 → 태스크 0 자동 생성 (학습 목표 아님)

---

## 태스크 0에 포함할 것

### 학습 목표와 무관한 것
- VPC, 서브넷, 인터넷 게이트웨이, NAT Gateway
- 라우팅 테이블, 보안 그룹 (기본 설정)
- IAM 역할 (Lambda 실행 역할 등)

### 시간이 오래 걸리는 것
- RDS 인스턴스 (5-10분)
- ElastiCache 클러스터 (3-5분)
- NAT Gateway (2-3분)

### 테스트용 리소스
- IAM 사용자 (Access Key 포함)
- 테스트 S3 버킷
- EC2 인스턴스 (네트워크 테스트용)

---

## 태스크 0에서 제외할 것

- ❌ 실습 제목에 명시된 리소스
- ❌ 학습 목표인 리소스
- ❌ 간단하고 빠른 리소스 (학습 목표인 경우)

---

## 태스크 번호 규칙

### 태스크 0 필요
- CloudFormation 템플릿 배포
- 사전 리소스 준비
- 실습 파일 다운로드

### 태스크 1부터 시작
- 사전 준비 없이 바로 시작 가능
- AWS 콘솔에서 직접 리소스 생성

---

## 판단 기준 요약

| 실습 제목 | 태스크 0 | 태스크 1-N |
|----------|---------|-----------|
| VPC Endpoint | VPC, 서브넷, EC2 | VPC Endpoint |
| 보안 그룹 | VPC, 서브넷 | 보안 그룹 |
| IAM 역할 | lab-user, S3 버킷 | IAM 역할 |
| Lambda | DynamoDB, IAM 역할 | Lambda 함수 |
| ElastiCache | VPC, RDS, EC2 | ElastiCache |
| DynamoDB | 없음 | DynamoDB 테이블 |
