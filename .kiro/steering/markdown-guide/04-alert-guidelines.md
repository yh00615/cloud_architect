# Alert 사용 가이드라인

실습 가이드에서 사용하는 10가지 Alert 타입과 사용 규칙입니다.

---

## Alert 타입 (10가지)

### 1. [!NOTE] - 참고 사항
- **색상**: 파란색 (info)
- **사용**: 대기 시간, 추가 정보, 비용 없음 안내
- **빈도**: 전체 5-8개 (최대 10개), 태스크당 1-2개 (최대 3개)
- **블릿 포인트**: 2개 이상 항목 시 사용, 앞에 빈 줄 필수

### 2. [!WARNING] - 경고
- **색상**: 노란색 (warning)
- **사용**: 리소스 삭제 경고, 비용 정보
- **위치**: Front Matter 직후, 리소스 정리 섹션

### 3. [!TIP] - 팁
- **색상**: 초록색 (success)
- **사용**: 모범 사례, 효율적 방법
- **빈도**: 전체 2-5개

### 4. [!SUCCESS] - 성공
- **색상**: 초록색 (status-positive 아이콘)
- **사용**: 태스크 완료, 성공 확인

### 5. [!ERROR] - 오류
- **색상**: 빨간색 (error)
- **사용**: 오류 메시지, 실패 상황

### 6. [!OUTPUT] - 예상 출력
- **색상**: 파란색
- **사용**: 명령어 실행 결과
- **필수**: 반드시 코드 블록 사용

### 7. [!IMPORTANT] - 중요 정보
- **색상**: 보라색
- **사용**: 필수 확인 사항, 건너뛰면 실패하는 단계

### 8. [!TROUBLESHOOTING] - 문제 해결
- **색상**: 주황색
- **사용**: 자주 발생하는 오류와 해결 방법
- **빈도**: 전체 1-2개

### 9. [!CONCEPT] - 개념 설명 (데모 전용)
- **색상**: 파란색
- **사용**: 기술 개념, 아키텍처 원리
- **데모**: 적극 활용 (태스크당 1-3개)
- **실습**: 필요시만 (복잡한 개념, 전체 1-3개)
- **연속 사용 금지**: Alert 사이에 다른 내용 삽입

### 10. [!DOWNLOAD] - 파일 다운로드
- **사용**: 실습 파일 제공
- **위치**: Front Matter 직후, WARNING 전
- **필수**: 파일 목록 + 관련 태스크

---

## 사용 빈도 가이드

### 데모 가이드 (총 10-15개)
- CONCEPT: 적극 활용
- NOTE: 태스크당 1-2개
- TIP: 전체 2-4개
- WARNING: 시작 + 리소스 정리

### 실습 가이드 (총 10-15개)
- CONCEPT: 필요시만 (전체 1-3개)
- NOTE: 전체 5-8개 (최대 10개)
- TIP: 전체 2-5개
- IMPORTANT: 필요시
- TROUBLESHOOTING: 전체 1-2개
- DOWNLOAD: 파일 제공 시
- WARNING: 시작 + 리소스 정리

---

## Alert 우선순위

1. WARNING (비용, 보안)
2. IMPORTANT (필수 확인)
3. CONCEPT (개념 이해)
4. NOTE (참고)
5. TIP (권장)
6. OUTPUT (결과)
7. TROUBLESHOOTING (문제 해결)

---

## 금지 사항

- ❌ Alert 총 개수 초과 (18개 이하)
- ❌ NOTE 과다 사용 (전체 10개, 태스크당 3개 이하)
- ❌ 한 태스크에 Alert 4개 이상
- ❌ 실습에서 CONCEPT 과다 (전체 3개 이하)
- ❌ 중요하지 않은 내용에 IMPORTANT
- ❌ 예상 출력을 이모지로 표시
- ❌ 일반 텍스트로 충분한 내용을 NOTE로

---

## NOTE Alert 사용 기준

### ✅ 사용해야 하는 경우
- 대기 시간 + 활용 제안
- 중요한 기술적 배경 (3줄 이상)
- 예상치 못한 동작
- 오류 가능성 사전 안내
- 상태 변화 상세 설명

### ❌ 사용하지 말아야 하는 경우
- 단순 설명 (1-2줄) → 일반 텍스트
- 짧은 확인 사항 → 일반 텍스트
- 당연한 정보 → 제거
- 단순 대기 시간만 → 일반 텍스트
