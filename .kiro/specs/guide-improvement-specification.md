## ❌ 개선이 필요한 부분

### 1. 태스크 0 누락 (모든 가이드)

**문제점:**
- 모든 가이드가 태스크 1부터 시작
- 복잡한 환경 구축을 수동으로 진행 (VPC, 서브넷, 보안 그룹 등)
- 학생이 환경 구축에 너무 많은 시간 소비

**예시 - 3-1-vpc-design-strategy.md:**
```markdown
## 태스크 1: VPC 생성 및 CIDR 블록 설계
## 태스크 2: Public 서브넷 생성
## 태스크 3: Private 서브넷 생성
## 태스크 4: Internet Gateway 생성 및 연결
## 태스크 5: Public 서브넷용 라우팅 테이블 생성
...
```

**개선 방향:**
```markdown
## 태스크 0: 실습 환경 구축

이 태스크에서는 CloudFormation을 사용하여 VPC, 서브넷, 라우팅 테이블 등 기본 인프라를 자동으로 생성합니다.

1. 다운로드한 `week3-1-vpc-lab.zip` 파일의 압축을 해제합니다.
2. `cloudformation-template.yaml` 파일을 확인합니다.
3. AWS Management Console에 로그인한 후 상단 검색창에서 `CloudFormation`을 검색하고 선택합니다.
...

## 태스크 1: VPC Endpoint 생성 (핵심 학습 목표)
## 태스크 2: 라우팅 테이블 업데이트
## 태스크 3: 동작 확인
```

### 3. 리소스 정리 복잡도 (3-1, 5-1)

**문제점 - 3-1-vpc-design-strategy.md:**
```markdown
## 리소스 정리

1. NAT Gateway 삭제
2. Elastic IP 해제
3. Internet Gateway Detach
4. Internet Gateway 삭제
5. 서브넷 삭제 (4개)
6. VPC 삭제
... (총 23단계)
```

**개선 방향:**
```markdown
## 리소스 정리

> [!WARNING]
> 다음 단계를 반드시 수행하여 불필요한 비용을 방지하세요.

1. CloudFormation 콘솔로 이동합니다.
2. `week3-1-lab-stack` 스택을 선택합니다.
3. [[Delete]] 버튼을 클릭합니다.
4. 확인 창에서 [[Delete]] 버튼을 클릭합니다.
5. 스택 삭제가 완료될 때까지 기다립니다 (2-3분 소요).

> [!NOTE]
> CloudFormation 스택을 삭제하면 VPC, 서브넷, IGW, NAT Gateway, Elastic IP 등 모든 리소스가 자동으로 삭제됩니다.

✅ **실습 종료**: 모든 리소스가 정리되었습니다.
```
