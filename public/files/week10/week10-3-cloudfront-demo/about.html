실습 목표

  • CloudFront 배포를 생성하고 S3 오리진을 연결할 수 있습니다
  • OAC(Origin Access Control)를 설정하여 S3 버킷을 보호할 수 있습니다
  • 캐싱 동작과 TTL 설정을 이해할 수 있습니다
  • 캐시 무효화(Invalidation)를 수행할 수 있습니다
  • CloudFront 메트릭을 모니터링할 수 있습니다

캐싱 테스트 파일 유형

HTML

index.html, about.html

TTL: 짧게 설정 (예: 300초)

CSS

style.css

TTL: 중간 (예: 3600초)

JavaScript

script.js

TTL: 중간 (예: 3600초)

이미지

logo.png, banner.jpg

TTL: 길게 설정 (예: 86400초)

CloudFront 모범 사례

  • 파일 타입별 TTL 설정: 정적 파일은 길게, 동적 콘텐츠는 짧게
  • 압축 활성화: Gzip/Brotli 압축으로 전송 속도 향상
  • HTTPS 사용: SSL/TLS 인증서로 보안 강화
  • OAC 사용: S3 버킷을 비공개로 유지하고 CloudFront를 통해서만 접근
  • 캐시 키 최적화: 쿼리 문자열, 헤더, 쿠키 설정 최적화
  • 모니터링: CloudWatch 메트릭으로 성능 추적

문제 해결

403 Forbidden 오류

원인: S3 버킷 정책이 CloudFront OAC를 허용하지 않음

해결: CloudFormation이 제안하는 버킷 정책을 복사하여 S3 버킷에 적용

오래된 콘텐츠가 표시됨

원인: 캐시가 아직 만료되지 않음

해결: CloudFront 콘솔에서 캐시 무효화(Invalidation) 생성

느린 첫 로딩

원인: 캐시 미스 (Miss from cloudfront)

정상: 첫 요청은 오리진에서 가져오므로 느릴 수 있음. 이후 요청은 빠름
